# Prompted Segmentation for Drywall QA — Technical Report

## 1\. Goal Summary

We train a **text-conditioned binary segmentation model** that accepts an RGB image and a natural-language prompt and outputs a single-channel binary mask (values {0, 255}) identifying:

* **"segment taping area"** — drywall joint/tape regions (Dataset 1)
* **"segment crack"** — wall crack defects (Dataset 2)

The model must generalise across varied scene appearances and be robust to alternative phrasings of the same prompt.

\---

## 2\. Approach

### Model: CLIPSeg

We select **CLIPSeg** (`CIDAS/clipseg-rd64-refined`) as our base model. CLIPSeg extends CLIP (ViT-B/16) with a lightweight transformer decoder that conditions on the CLIP text embedding of the prompt to produce a segmentation logit map. This makes it well-suited for text-prompted segmentation without architectural changes for new class labels.

**Architecture:**

* Text encoder: CLIP ViT-B/16 text branch (frozen during fine-tuning)
* Vision encoder: CLIP ViT-B/16 vision branch (fine-tuned)
* Decoder: 3-block transformer conditioned on text embedding → 64×64 logit map
* Upsampling: bilinear to original image size at inference time

**Alternative models considered:**

|Model|Verdict|
|-|-|
|SAM (Segment Anything)|Prompt is spatial (point/box), not text — needs adapter|
|SEEM / X-Decoder|Larger, more VRAM, slower iteration|
|GroundedSAM|Two-stage; adds latency; no end-to-end mask fine-tuning|
|CLIPSeg|✅ Native text prompts, \~68M params, fast fine-tune|

### Loss Function

```
L = 0.5 × BCE(σ(logits), GT) + 0.5 × DiceLoss(σ(logits), GT)
```

BCE penalises per-pixel errors. Dice is class-imbalance-robust — critical for crack datasets where foreground pixels are sparse (<5% of image area).

Splits are provided by Roboflow and preserved as-is (scene-level independence maintained). Annotations are COCO Segmentation polygons rasterised to binary PNG masks.

**Prompt augmentation** — during training, each sample randomly samples one of 3–5 prompt phrasings per class to improve prompt robustness.

\---

## 4\. Metrics

|Metric|Definition|
|-|-|
|**mIoU**|mean(Intersection / Union) per image, averaged over test set|
|**Dice**|mean(2·TP / (2·TP + FP + FN)) per image, averaged over test set|

Both computed at threshold = 0.5 on sigmoid-activated logits upsampled to original resolution.

## 7\. Failure Analysis

|Failure mode|Frequency|Root cause|
|-|-|-|
|Under-segmented thin cracks|Moderate|64×64 logit map cannot resolve hairline cracks|
|Missed painted-over tape|Low–moderate|Low contrast; CLIP features indistinguishable from plain wall|
|False positives at corners|Low|CLIP vision confuses sharp geometry with crack patterns|
|Prompt-sensitivity inconsistency|Rare|Uncommon phrasings less represented in CLIP pretraining|

**Mitigations applied:** prompt augmentation, BCE+Dice combined loss, colour jitter augmentation.
